# 命名・フォーマット仕様

エージェント定義・スキル定義・ポリシー文書のファイル命名規則と構造仕様の正規リファレンスです。

## ファイル命名規則

| 対象 | パス | 命名規則 | 例 |
|------|------|---------|-----|
| エージェント | `.github/agents/<name>.md` | kebab-case | `agent-builder.md`, `code-reviewer.md` |
| スキル | `.github/skills/<name>/SKILL.md` | kebab-case ディレクトリ + 固定ファイル名 `SKILL.md` | `ci-hygiene/SKILL.md` |
| ポリシー | `.github/policies/<name>.md` | kebab-case | `ci-tools.md`, `repository-checklist.md` |
| エージェントドキュメント | `agent_docs/<name>.md` | kebab-case | `reference-map.md` |
| 人間向けドキュメント | `human_docs/<name>.md` | kebab-case | `getting-started.md` |

### 命名の原則

- **kebab-case を厳守**: 小文字英字とハイフンのみ。アンダースコア・キャメルケース・大文字は不可
- **簡潔で意図が明確**: 2〜3 単語で役割を表現する
- **動詞+名詞 または 名詞+修飾語**: `code-reviewer`（名詞+名詞）、`git-workflow`（名詞+名詞）

```text
# 正例
agent-builder.md
dev-env-setup/SKILL.md
repository-checklist.md

# 反例
AgentBuilder.md        ← キャメルケース不可
dev_env_setup/SKILL.md ← アンダースコア不可
CI-Tools.md            ← 大文字不可
```

## エージェント定義の構造

### コードフェンス

エージェント定義は `` ```chatagent `` コードフェンスで囲む。他のフェンス（`` ```markdown `` 等）は不可。

### フロントマター

YAML フロントマターで以下のフィールドを定義する:

| フィールド | 必須 | 型 | 説明 |
|-----------|------|-----|------|
| `name` | 必須 | string | エージェント名（kebab-case） |
| `description` | 必須 | string | 1 行の説明（ダブルクォートで囲む） |
| `tools` | 必須 | list | 使用ツールのリスト |

#### `tools` に指定可能な値

| ツール | 用途 |
|--------|------|
| `search` | ワークスペース内のファイル検索（全エージェントで推奨） |
| `editFiles` | ファイルの作成・編集 |
| `runInTerminal` | ターミナルコマンドの実行 |
| `fetch` | 外部 URL からのデータ取得 |
| `vscode/askQuestions` | ユーザーへの質問・確認（計画を含むエージェントで推奨） |

### 本文の見出し構造

以下の順序で見出しを配置する。`## TDD 実装サイクル` はコード実装エージェントのみ。

```text
# <タイトル> — <サブタイトル>

あなたは **<Agent Name>** です。<ロール説明>

## 行動原則              ← 3〜5 項目の番号付きリスト
## ヒアリング項目        ← 対話生成型のみ（任意）
## チェックリスト        ← タスクリスト形式（任意）
## 作業手順              ← Step 形式で記述（必須）
## 出力フォーマット      ← 読取専用型のみ（任意）
## TDD 実装サイクル      ← コード実装エージェントのみ（任意）
## 禁止事項              ← 箇条書き（必須）
```

### 完全な例

```text
---
name: example-agent
description: "例示用のエージェント定義"
tools:
  - search
  - editFiles
---

# Example Agent — 例示エージェント

あなたは **Example Agent** です。具体的な役割をここに記述します。

## 行動原則

1. **原則名**: 説明
2. **原則名**: 説明
3. **原則名**: 説明

## 作業手順

### Step 1: タスク名

手順の詳細をここに記述する。

### Step 2: タスク名

手順の詳細をここに記述する。

## 禁止事項

- 禁止事項 1
- 禁止事項 2
```

## スキル定義の構造

### フロントマター

| フィールド | 必須 | 型 | 説明 |
|-----------|------|-----|------|
| `name` | 必須 | string | スキル名（kebab-case） |
| `description` | 必須 | string | 1 行の説明（ダブルクォートで囲む） |

### 本文の見出し構造

```text
# <スキルタイトル>

## 概要                  ← スキルの目的と適用範囲
## 手順 / テンプレート    ← Step 形式のテーブルまたはコードブロック
## 関連スキル            ← 他スキルへのパス参照（任意）
```

### 完全な例

```text
---
name: example-skill
description: "例示用のスキル定義"
---

# Example Skill

## 概要

このスキルの目的と適用範囲を記述する。

## 手順

| Step | 操作 | 詳細 |
|------|------|------|
| 1 | 準備 | 前提条件を確認する |
| 2 | 実行 | メインの操作を実行する |
| 3 | 検証 | 結果を確認する |

## テンプレート

設定ファイルのテンプレートをコードブロックで記載する。

## 関連スキル

- `/related-skill` — `.github/skills/related-skill/SKILL.md`
```

## ポリシー文書の構造

### フォーマット

- コードフェンスやフロントマターは**使用しない**
- 通常の Markdown として記述する

### 見出し構造

```text
# <ポリシータイトル>

冒頭に目的と適用範囲の説明。

## 必須項目              ← チェックリスト形式
### カテゴリ名
- [ ] **項目名**: 説明

## 推奨項目              ← チェックリスト形式（任意）
## 更新履歴              ← テーブル形式（任意）
```

### 記法パターン

```markdown
# ポリシーの項目は定義リスト風に記述する

- **項目名**: 説明文
- **CI ワークフロー**: PR に対して自動チェックが実行される
```

## バリデーションチェックリスト

新規ファイル作成時に以下を確認してください:

### エージェント定義

- [ ] ファイル名が kebab-case（`.github/agents/<name>.md`）
- [ ] `` ```chatagent `` コードフェンスで囲まれている
- [ ] フロントマターに `name`, `description`, `tools` が存在する
- [ ] `name` がファイル名（拡張子除く）と一致する
- [ ] `description` がダブルクォートで囲まれている
- [ ] 本文に `## 行動原則`, `## 作業手順`, `## 禁止事項` が存在する
- [ ] コード実装エージェントの場合、`## TDD 実装サイクル` が存在する

### スキル定義

- [ ] ディレクトリ名が kebab-case、ファイル名が `SKILL.md`
- [ ] フロントマターに `name`, `description` が存在する
- [ ] `name` がディレクトリ名と一致する
- [ ] 本文に `## 概要` が存在する

### ポリシー文書

- [ ] ファイル名が kebab-case（`.github/policies/<name>.md`）
- [ ] コードフェンスやフロントマターが**ない**
- [ ] `# タイトル` で始まっている
