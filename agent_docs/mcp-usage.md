# MCP サーバー利用ガイド

`.vscode/mcp.json` で定義された MCP（Model Context Protocol）サーバーの用途と利用パターンです。

## 利用可能なサーバー

| サーバー | 接続方式 | 主な用途 |
|---------|---------|---------|
| GitHub MCP Server | Docker（stdio） | GitHub リソース（Issue・PR・リポジトリ）の操作 |
| Context7 MCP Server | npx（stdio） | ライブラリ・フレームワークの公式ドキュメント取得 |

---

## GitHub MCP Server

### 概要

GitHub の公式 MCP サーバーです。リポジトリ・Issue・PR・ブランチなどの GitHub リソースを
プログラマティックに操作できます。

### 認証

- Docker コンテナ内で `GITHUB_PERSONAL_ACCESS_TOKEN` 環境変数を使用
- 必要なスコープ: `repo`, `read:org`（推奨）
- トークンは `.vscode/mcp.json` の `inputs` で起動時にプロンプト入力される

### 主な利用シナリオ

| シナリオ | 操作例 | 利用するエージェント |
|---------|--------|-------------------|
| PR 作成 | ブランチの差分から PR を作成 | `@repo-guardian` |
| PR 差分取得 | レビュー対象の PR のファイル変更を取得 | `@code-reviewer` |
| Issue 参照 | 関連 Issue の内容を確認 | 全エージェント |
| リポジトリ情報 | デフォルトブランチ・設定の確認 | `@repo-guardian` |

### 利用時の注意

- **使うべき場面**: GitHub API でしか取得できない情報（PR 差分・Issue コメント・リポジトリ設定）が必要な場合
- **使わない場面**: ローカルのファイル操作（`search` / `editFiles` で対応可能な場合）
- **認証エラー時**: トークンが未設定または期限切れの場合、ユーザーに再認証を促す

## Context7 MCP Server

### 概要

ライブラリやフレームワークの**最新の公式ドキュメント**を取得するための MCP サーバーです。
エージェントが技術的な判断を行う際に、正確なドキュメント情報を参照できます。

### 認証

- 認証不要（公開ドキュメントの取得のみ）

### 主な利用シナリオ

| シナリオ | 操作例 | 利用するエージェント |
|---------|--------|-------------------|
| Dev Container 構成 | 言語・フレームワークの推奨設定を確認 | `@dev-env-builder` |
| CI 設定 | テストフレームワークの実行コマンドを確認 | `@dev-env-builder` |
| コードレビュー | ライブラリの API 仕様を確認 | `@code-reviewer` |
| エージェント設計 | ツール・フレームワークのベストプラクティスを調査 | `@agent-builder` |

### 利用時の注意

- **使うべき場面**: 特定のライブラリ・フレームワークの設定方法や API 仕様を正確に確認したい場合
- **使わない場面**: リポジトリ内のファイル情報で十分な場合（ローカル検索を優先する）
- **制限事項**: 一部のマイナーなライブラリはドキュメントが利用できない場合がある

## エージェント別の推奨利用パターン

### `@agent-builder`

```text
Context7 → 指定された技術スタックのベストプラクティスを調査
         → 生成するエージェントに適切な技術的知識を組み込む
```

### `@dev-env-builder`

```text
Context7 → フレームワークの推奨 Dev Container 設定を確認
         → テストフレームワークの実行コマンドを検証
         → リンター・フォーマッターの設定オプションを参照
```

### `@repo-guardian`

```text
GitHub MCP → リポジトリの現在の設定・ブランチ保護ルールを確認
           → 監査結果に基づく PR を作成
           → 既存の Issue/PR と重複がないか確認
```

### `@code-reviewer`

```text
GitHub MCP → PR の差分・コメント・レビュー履歴を取得
Context7   → 使用ライブラリの API 仕様を確認（必要時のみ）
```

### `@docs-writer`

```text
GitHub MCP → PR の変更内容を確認（ドキュメント更新の根拠として）
```

## MCP サーバー設定の場所

設定ファイル: `.vscode/mcp.json`

MCP サーバーの追加・変更を行う場合は、このファイルを編集してください。
新しいサーバーを追加する際は、コメントで用途と認証方法を記載してください。
